## Important caveat

A green baseline here does **not** mean the code is correct. The suite passes because it does not
yet test the things the blueprint identifies as broken. Specifically, none of the 13 tests
covers:

- float RGB in PLY being divided by 255 (**P0-07**);
- writers targeting the final path directly and destroying an existing file on failure
  (**P0-08**);
- SPZ v4 containers at all (**P0-09**);
- `KHR_gaussian_splatting` (**P0-10**);
- resource exhaustion from large well-formed input (**P0-12**);
- position-only scene transforms leaving covariance and SH untouched (**P0-17**).

"The tests pass" is evidence about the tests, not about the software. The first act of most of
the following work packages is to add the failing test that this baseline is missing.
