Material improvements

Native input and math

  • Bounded, alignment-safe glTF accessor reads with finite/configuration limits.
  • Active-scene traversal, matrix/TRS composition, instancing, cycle protection, and inverse-transpose normals.
  • Robust quaternions, log-scale covariance, little-endian PLY, and CPU/Metal/CUDA reference alignment.
  • Correct front-to-back compositing, final-transmittance background, and backward recurrence.

Honest product surface

  • Native fit/feedforward commands now fail closed instead of reporting success for incomplete implementations.
  • Unsafe DA3 view sharding retired; one jointly inferred scene retains a shared camera frame.
  • CoreML CLI limited to the two experimentally supported commands.
  • Documentation no longer advertises unsupported speed, benchmark, format, or parity claims.

ML and model supply chain

  • Official DA3 source/checkpoint revisions are immutable, validated, marked, and installed atomically.
  • Depth conversion uses camera-Z geometry and rejects missing/malformed cameras and empty output.
  • Remote-code trust, executable eval/shell construction, unsafe pickle, and unrestricted model loads removed from exposed paths.
  • Unmaintained research snapshots carry explicit quarantine notices.

Viewer and distribution

  • Bounded circular 4D prefetch, latest-seek wins, finite retries, recovery states, and reduced-motion/mobile behavior.
  • GET/HEAD allowlist server, restrictive Tauri CSP, context-loss recovery, and safe DOM updates.
  • Deterministic project-owned demos and exact digest-checked staging exclude unlicensed sample scenes.
  • Project, Spark, Three, and locked Rust dependency notices ship with the desktop payload.

CI and governance

  • Immutable action SHAs, least privilege, concurrency cancellation, and bounded job timeouts.
  • macOS Metal/sanitizer, Linux CPU/CUDA compile, Python/shell, viewer, Rust, Tauri, CoreML, and secret-scan jobs.
  • Dependabot, dependency review, CODEOWNERS, cargo-deny, npm audit, and frozen license inventory.
  • Install layout includes project and third-party notices; cross-manifest version drift is tested.

Frontend craft

  • Responsive controls, keyboard/focus behavior, aria-live status, touch targets, and safe-area handling.
  • Loading, retry, unsupported-browser, and WebGL-context states are visible and recoverable.
  • Runtime allocations reduced with scratch objects and production rendering flags.
  • Playwright locks behavior across desktop, mobile, static, and temporal scenes.

Scope and interpretation

This was a repository-wide engineering review of maintained first-party paths and the vendored code they expose. It included source inspection, adversarial boundary tests, local builds, browser automation, package policy, provenance, and documentation/executable agreement.

It was not a new research-paper evaluation. Newer reconstruction methods are tracked in a dated catalog, but none is described as integrated or superior without a model-correct adapter and reproducible evidence.

Review limitations
  • No NVIDIA GPU was attached to the review host.
  • No signing identities, notarization account, or external release authority were in scope.
  • No large proprietary model weights or benchmark datasets were downloaded.
  • Remote CI must still run on the committed tree; local actionlint cannot emulate every hosted runner detail.
