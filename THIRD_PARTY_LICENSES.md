Melkor's core is MIT-licensed. This file records every third-party component the
repository depends on, and distinguishes what Melkor **redistributes** from what it
merely **invokes**. That distinction is the whole point: shipping a library means its
licence travels with our artifact, whereas calling a program the user installed does
not.

### Modifications to spz

Melkor carries local patches against upstream `2cf0be04d86b`. They are
kept as reviewable files under `third_party/patches/spz/` rather than
edited into the vendored source, so that the fork is visible and upstream can be
upgraded deliberately.

- **`0001-probe-spz-version-without-inflating.patch`** — Upstream offers no way to read a file's format version without first inflating its gzip payload, which forces decompression of attacker-controlled data before any version policy check can run. Adds a bounded header-only probe.
  Upstream status: not-submitted.
- **`0002-bound-fixed-point-bits-and-finite-logits.patch`** — fractionalBits is read from the file and used as a shift exponent with no upper bound (max is 23 for signed 24-bit fixed point). Separately, invSigmoid maps packed alpha bytes 0 and 255 to infinite logits, so even a default opacity of 1.0 round-trips to a non-finite value. Both are clamped.
  Upstream status: not-submitted.

## Invoked but not redistributed

Reconstruction, training, and depth systems are separate programs. Melkor describes
them with pinned adapter manifests; it does not vendor their source or their weights.
Their code licences and their **model-weight licences are different fields**, because
they routinely differ — permissive code frequently ships with research-only weights.

The upstream projects, their licences, and why the research snapshots were removed
from this repository are documented in [docs/adapters/index.md](docs/adapters/index.md).

## Viewer dependencies

The web viewer's npm dependencies and the Tauri desktop shell's Rust crates are
locked in `viewer/package-lock.json` and `viewer/src-tauri/Cargo.lock`. The desktop
shell generates its own crate licence inventory
(`viewer/RUST_THIRD_PARTY_LICENSES.html`) via `cargo-about`, and `cargo-deny`
enforces the licence policy in CI.
